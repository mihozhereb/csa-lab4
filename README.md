## Язык программирования - Forth

### Описание синтаксиса

``` ebnf
<program> ::= <terms>

<terms> ::= { <term> }

<term> ::= <number>
         | <operand>
         | <identifier>
         | <var_init>
         | <word_init>
         | <if-term>
         | <loop-term>
         | <import-term>
         | <comment>

<number> ::= <знаковое десятичное число>
<string> ::= s"<любая последовательность символов>"

<operand> ::= "+"
            | "-"
            | "/"
            | "*"
            | "&"
            | "|"
            | "^"
            | "~"
            | "dup"
            | "drop"
            | "swap"
            | "over"
            | "@"
            | "!"

<identifier> ::= <letter> { <letter> | <digit> | "_" }

<var_name> ::= <identifier>
<var_init> ::= "VAR" <number> <var_name>
             | "VAR" <string> <var_name>
             | "VAR" "ALLOC" <number> <var_name>

<word> ::= <identifier>
<word_init> ::= ":" <word> <terms> ";"

<if-term> ::= "IF" <terms> "THEN"
            | "IF" <terms> "ELSE" <terms> "THEN"

<loop-term> ::= "BEGIN" <terms> "WHILE" <terms> "REPEAT"

<import-term> ::= "INCLUDE" <string>

<comment> ::= "\" <любая последовательность символов до переноса строки>
            | "(" <любая последовательность символов> ")"
```

### Описание семантики

- Язык является стековым. Все операции выполняются над стеком данных. Литералы при выполнении помещаются на вершину стека, арифметические и логические операции берут операнды с вершины стека и помещают результат обратно.

- Код выполняется последовательно за исключением процедур. Вызов процедуры осуществляется при указании в программе её названия.

- Используется постфиксная запись (обратная польская нотация).
Например:
`
2 3 +
`
сначала помещает 2 и 3 на стек, затем операция + снимает два значения и кладёт результат 5.

- Оператор объявления переменной `VAR` создаёт переменную и сопоставляет ей определенную ячейку памяти. При вызове `VAR_NAME` адрес переменной кладется на стек, с адресом можно взаимодействовать при помощи команд `!` и `@`. В случае добавления ключевого слова `ALLOC` выделяется непрерывный участок памяти с фиксированным в программе размером.

- Условные конструкции используют значение на вершине стека: 0 считается ложью, любое ненулевое значение -- истиной.

- Код между `BEGIN` и `WHILE` вычисляет условие и должен оставить на вершине стека флаг.
Если флаг равен 0, цикл завершается. Если флаг не равен 0, выполняется тело цикла между `WHILE` и `REPEAT`. После выполнения тела управление снова переходит к `BEGIN`.

- Файлы можно импортировать друг в друга. Например: `"math.fs" INCLUDE`. Импорт является inline вставкой текста из импортируемого файла.

### Области видимости

- Имена переменных и слов хранятся в общем словаре языка.

- После объявления слово или переменная доступны во всём коде, который расположен после объявления.

- Если имя объявляется повторно, новая запись перекрывает старую.

- Локальных областей видимости нет.

### Типизация, виды литералов

- Слабая динамическая типизация. Память выделяется статически при компиляции.

- Существует два вида литералов: знаковые целочисленные литералы и строковые литералы.

- Строковые литералы объявляются как `VAR s"..." VAR_NAME` или `VAR ALLOC <размер строки> VAR_NAME`. В обоих случаях с `VAR_NAME` связывается адрес начала секции, выделенной под строку.

- Строки представляются как `CSTR`, окончания строки обозначается символом `\0`
